Survival Analysis

Breast Cancer Data: Kaplan-Meier, Exponential and Weibull Models

Published

March 29, 2026

Overview

This script demonstrates survival analysis techniques for time-to-event data where some observations may be censored (we don’t observe the full time until the event). We’ll estimate survival curves non-parametrically (Kaplan-Meier) and then fit parametric survival models (exponential and Weibull).

Key concept: In survival analysis, we model the time until an event occurs (death, relapse, etc.), accounting for censoring (incomplete observations due to study end or loss to follow-up).

ImportantSELF TEST: What does the Kaplan-Meier curve estimate?

The Kaplan-Meier curve estimates \(\hat{S}(t)\) = the probability that a randomly selected subject survives beyond time \(t\). It’s non-parametric (makes no distributional assumptions) and handles censoring by using “risk sets”—the number still at risk at each event time. Events that occur, combined with censored observations, inform the survival probabilities.

Weibull Suitability Check

Before fitting parametric models, check if the Weibull distribution is suitable. The log-log transformation of a Weibull survival curve is linear: \(\log(-\log(S(t))) = \alpha \log(\lambda t)\)

Tip

Diagnostic plot interpretation: - Linear relationship: Suggests Weibull fit is reasonable - Curved pattern: Suggests Weibull may not fit; consider exponential or other models

Code 
opar <- par(mar = c(4, 4, 0, 0), las = 1)
negative <- bcancer$group == "N"
positive <- bcancer$group == "P"

Proportional Hazards Assumption

Many survival models (e.g., Cox) assume proportional hazards: the hazard ratio between groups is constant over time. Check this with the cloglog transformation.

Note

Proportional hazards assumption: If the cloglog curves are parallel, hazards are proportional. If curves cross, the assumption is violated, suggesting the treatment effect changes over time.

Code 
opar <- par(mar = c(2, 2, 0, 0), las = 1)

# cloglog transformation: log(-log(S(t)))
plot(
    cancer.km,
    fun = "cloglog",
    lty = 1:2,
    col = c("Red", "Blue"),
    xlim = c(5, 300)
)

legend(
    "topleft",
    legend = c("Negative Stain", "Positive Stain"),
    lty = 1:2,
    col = c("Red", "Blue"),
    bty = "n"
)
Code 
par(opar)
ImportantSELF TEST: What does parallelism of cloglog curves tell us?

Parallel cloglog curves suggest the proportional hazards assumption holds: the hazard of death for positive-stain patients is a constant multiple of that for negative-stain patients, regardless of time. Crossing curves suggest the groups’ relative hazards change over time.

Exponential Regression

The exponential model is a parametric survival model with constant hazard rate. It assumes failures follow a Poisson process.

Fit exponential survival model:

Code 
cancer_exp <- survreg(
    Surv(time, status) ~ group,
    data = bcancer,
    dist = "exponential"
)

summary(cancer_exp)

Exponential model assumptions: Constant hazard rate (hazard doesn’t change over time). This is restrictive and often violated in practice, but is a useful baseline and computationally simple.

Testing the Group Effect

Wald Test:

Code 
wald_stat <- cancer_exp$coeff[2] / sqrt(cancer_exp$var[2, 2])
2 * pnorm(-abs(wald_stat))
    groupP 
0.05582701

This gives a p-value for whether group significantly affects survival time.

Likelihood Ratio Test:

Code 
lr_stat <- -2 * (cancer_exp$loglik[1] - cancer_exp$loglik[2])
pchisq(q = lr_stat, df = 1, lower.tail = FALSE)
[1] 0.03687179

An alternative p-value based on the likelihood ratio statistic.

ImportantSELF TEST: When would Wald and LR tests disagree?

Both test the same hypothesis but use different asymptotics. For moderate sample sizes, they usually agree. With small samples or if the coefficient is near a boundary (e.g., near no effect), they can diverge. LR tests are generally more robust in finite samples.

Extracting Rate Parameters

For the exponential model, the rate parameter (hazard) is \(\lambda = e^{(intercept + \beta \cdot \text{group})}\):

Code 
lambda_n <- exp(-cancer_exp$coef[1])
lambda_p <- exp(-sum(cancer_exp$coef))

Note

Negative stain group: \(\lambda_N =\) 0.0030266 events/time
Positive stain group: \(\lambda_P =\) 0.0078387 events/time

Higher \(\lambda\) means shorter survival (higher hazard rate).

Verification: MLE via Method of Moments

For the exponential distribution, the MLE is: \(\hat{\lambda} = \frac{\text{# events}}{\text{total time at risk}}\)

Code 
sum(bcancer$status[bcancer$group == "N"]) /
    sum(bcancer$time[bcancer$group == "N"])
[1] 0.003026634
Code 
sum(bcancer$status[bcancer$group == "P"]) /
    sum(bcancer$time[bcancer$group == "P"])
[1] 0.007838746
ImportantSELF TEST: Do these match the model rates above?

These hand-computed values should match (to numerical precision) the rates extracted above. This verifies that survreg is fitting via maximum likelihood and our interpretation of the rates is correct.

Delta Method Confidence Interval

Construct a 95% CI for \(\lambda_P\) using the delta method (linearization):

Code 
beta <- cancer_exp$coeff[2]
var_beta <- (cancer_exp$var)[2, 2]

# Delta method: var(e^X) ≈ (e^X)^2 * var(X)
var_lambda <- (exp(-beta))^2 * var_beta

# CI
exp(-beta) + qnorm(0.975) * sqrt(var_lambda)
  groupP 
5.115883
Code 
exp(-beta) - qnorm(0.975) * sqrt(var_lambda)
    groupP 
0.06396006
Note

The lower bound is positive (doesn’t cross zero), which is good since \(\lambda\) must be positive. However, the delta method doesn’t always respect parameter constraints; a safer approach is to transform:

Better approach: Transform to enforce positivity:

Code 
# Build CI on log scale, then exponentiate
exp(-(beta - qnorm(0.975) * sqrt(var_beta)))
  groupP 
6.868405
Code 
exp(-(beta + qnorm(0.975) * sqrt(var_beta)))
   groupP 
0.9766015
Note

This guarantees both bounds are positive because exponentiation preserves the ordering and always produces positive values.

Weibull Regression

The Weibull model generalizes the exponential by allowing a shape parameter \(\alpha > 0\). When \(\alpha = 1\), it reduces to exponential; \(\alpha > 1\) means hazard increases with time (wear-out); \(\alpha < 1\) means hazard decreases (infant mortality).

Fit Weibull survival model:

Code 
cancer_wbl <- survreg(
    Surv(time, status) ~ group,
    data = bcancer,
    dist = "weibull"
)

Note: survreg parameterizes the scale as \(\sigma = 1/\alpha\), so the shape parameter is \(\alpha = 1/\text{scale}\). A scale close to 1 means \(\alpha \approx 1\) (nearly exponential); scale >> 1 means \(\alpha << 1\) (rapidly decreasing hazard).

Testing the Shape Parameter

Wald test for whether shape = 1 (exponential):

Code 
wald_z <- log(cancer_wbl$scale) / sqrt(cancer_wbl$var[3, 3])

# p-value for H0: log(scale) = 0, i.e., scale = 1, i.e., alpha = 1
2 * pnorm(-abs(wald_z))
[1] 0.6993412

ImportantSELF TEST: Does the Weibull model fit better than exponential?

Compare the LR test p-value: if p < 0.05, the shape parameter is significantly different from 1, meaning Weibull provides better fit than exponential. Also compare AIC/BIC or look at the scaled deviances to assess relative model quality.

Key Learning Points

  1. Survival data structure: Time-to-event (\(t\)) and censoring indicator (0/1). Censoring indicates incomplete observation—we know \(T > t\) but not the exact value.

  2. Non-parametric vs. parametric:

    • Kaplan-Meier: Non-parametric, makes no distributional assumption, handles censoring via risk sets
    • Parametric models: Assume specific distribution (exponential, Weibull), more efficient if assumption holds

  3. Diagnostic plots:

    • Log-log plot: Linearity suggests Weibull fit is appropriate
    • cloglog plot: Parallelism checks proportional hazards assumption

  4. Model parameters:

    • Exponential: Single rate parameter \(\lambda\) (constant hazard)
    • Weibull: Rate \(\lambda\) and shape \(\alpha\) (hazard can increase/decrease over time)

  5. Interpretation:

    • Hazard ratio: relative risk between groups
    • Survival probability at time \(t\): probability of not having event by time \(t\)
    • Rates are on the inverse scale: higher rate = shorter survival time

  6. Inference:

    • Wald tests: Test individual parameters (less robust in finite samples)
    • LR tests: Compare nested models (more robust)
    • Always verify with bootstrapping or sensitivity analyses when possible
