Conditional Models: Happiness Data

Fahrmeir and Tutz (2001) Example 3.12

Published

March 25, 2026

Overview

We will fit a simple asymmetric model to a dataset on happiness levels from Table 3.10 of Fahrmeir and Tutz (2001). We want to determine the association between gender, level of education, and happiness levels. We will do this in two parts, first by setting gender as a predictor for years of schooling, and then by setting years of schooling as a predictor for happiness levels.

The model is specified more formally in the equation (3.5.3) in Fahrmeir and Tutz (2001).

Part 1: Marginal Distribution of School Education Levels by Sex

We want to fit a cumulative logit model to the marginal distribution of schooling, predicted by gender. To do this, we first need to arrange the data into wide form.

Code 
happy_wide <- happy_tbl |>
  mutate(
    School = factor(
      School,
      levels = c("<12", "12", "13-16", ">16"),
      labels = c("school1", "school2", "school3", "school4")
    )
  ) |>
  summarize(n = sum(n), .by = c(Sex, School)) |>
  pivot_wider(
    names_from = School,
    values_from = n,
    values_fill = 0
  )

happy_wide
# A tibble: 2 × 5
  Sex     school1 school2 school3 school4
  <fct>     <int>   <int>   <int>   <int>
1 Males       253     198     181      57
2 Females     304     309     183      33
TipUnderstanding the tidying
  • mutate() recodes factor levels so the column names match the model inputs.
  • summarize(..., .by = ...) aggregates counts within each group.
  • pivot_wider() spreads groups into columns and fills missing combinations with 0.

Setting Effect Coding

Effect coding is needed to reproduce the numbers in the book:

Code 
options("contrasts")
$contrasts

"contr.treatment"      "contr.poly"

contr.treatment is the default (corresponding to dummy coding), but we need contr.sum for effect coding:

Code

Females     0
Males       1
Code 
options(contrasts = c("contr.sum", "contr.poly"))

        [,1]
Females    1
Males     -1

We now have effect coding, however we need to switch the reference group to female, in order to match the book.

Code 
contrasts(happy_wide$Sex) <- as.matrix(c(-1, 1))
contrasts(happy_wide$Sex)

Females   -1
Males      1

Fit First Model

Now we can fit the model with school1 as the reference level.

Code 
happy_logit1_vglm <- vglm(
  cbind(school1, school2, school3, school4) ~ Sex,
  data = happy_wide,
  family = cumulative()
)

We now fit the model.

This regression ignores the Sex column (which is what we want).

  family = cumulative

When we compare the coefficients here to those in Table 3.11, only the two intercept estimates (corresponding to reference level <12 of School) match the numbers in the book. This is because under the dummy coding scheme, we are picking less than 12 years of school as our reference level. In contrast, the book treats each school intercept as it’s own absolute level (see second line of (3.5.3) in Fahrmeir and Tutz (2001)).

Reproducing the Deviance

We now attempt to reproduce the deviance.

To retrieve the likelihood of the full GLM model, we sum the log likelihood resulting from the first and second parts of the simple regressive cumulative model.

Computing Saturated Model Likelihood

Now we just need to compute the log-likelihood of the saturated model. We split the dataset into a male group and female group. Within each group we have 12 observations corresponding to the 12 combinations of schooling and happiness levels (meaning we have 11 degrees of freedom for the male and female groups separately). The maximum likelihood estimators for each group are exactly the observed proportions of each happiness level within each group.

Therefore we can calculate the log-likelihood of the saturated model by summing the log-likelihoods as follows:

Code 
happy_male <- dplyr::filter(happy, Sex == "Males")
happy_female <- dplyr::filter(happy, Sex == "Females")

log_lik_saturated <- dmultinom(
  happy_male$n,
  prob = happy_male$n / sum(happy_male$n),
  log = TRUE
) +
  dmultinom(
    happy_female$n,
    prob = happy_female$n / sum(happy_female$n),
    log = TRUE
  )

log_lik_saturated
[1] -57.99938

Deviance

Finally, we can compute the deviance as twice the difference between the log-likelihood of the saturated model and the log-likelihood of the fitted model:

Code 
2 * (log_lik_saturated - log_lik_glm)
[1] 13.27614

Problems with the Reported Standard Errors and \(p\)-values

This bit was slightly unclear to me, Dennis to clarify if I have misunderstood.

The second part of the the table are treating the model as by treating the years at school as fixed, and fitting a GLM to the data. The standard errors for are hence derived from the Fisher information matrix under the assumption that years at school are fixed. If we really want to understand the model as a whole, we need to treat both \(Y_1\) and \(Y_2\) as random. In practice this is probably very difficult to do, as we need to tease out the explicit form for the main function and so on.
