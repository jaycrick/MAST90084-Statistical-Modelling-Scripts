Ordinal Regression: Happiness Data

Cumulative Link Models for Ordered Outcomes

Published

March 23, 2026

Overview

This script demonstrates ordinal (cumulative link) regression on the happiness dataset, analyzing ordered categorical responses.

library(conflicted)
library(dplyr)
library(Fahrmeir)
library(tidyr)
library(VGAM)

Part 1: School Education Levels by Sex

Arrange the data into wide form for the first regression part in (3.5.3):

Code 
happy_wide <- happy |>
  as_tibble() |>
  mutate(
    School = factor(
      School,
      levels = c("<12", "12", "13-16", ">16"),
      labels = c("school1", "school2", "school3", "school4")
    )
  ) |>
  summarize(n = sum(n), .by = c(Sex, School)) |>
  pivot_wider(
    names_from = School,
    values_from = n,
    values_fill = 0
  )

happy_wide
# A tibble: 2 × 5
  Sex     school1 school2 school3 school4
  <fct>     <int>   <int>   <int>   <int>
1 Males       253     198     181      57
2 Females     304     309     183      33
TipUnderstanding the tidying
  • as_tibble() makes the data print-friendly and consistent with tidyverse tools.
  • mutate() recodes factor levels so the column names match the model inputs.
  • summarize(..., .by = ...) aggregates counts within each group.
  • pivot_wider() spreads groups into columns and fills missing combinations with 0.

Fit First Model

Regression for the first line of display (3.5.3) in F&T: To reproduce the first regression in section 3.5.3 of F&T, we cannot use polr() since it doesn’t allow for category-specific beta coefficients.

Code 
happy_logit1_vglm <- vglm(
  cbind(school1, school2, school3, school4) ~ Sex,
  data = happy_wide,
  family = cumulative()
)

Fit Second Model

Regression for the second line of display (3.5.3) in F&T:

Code 
happy_logit2_vglm <- vglm(
  cbind(happy_low, happy_medium, happy_high) ~ School,
  data = happy2,
  family = cumulative(link = "logitlink")
)

Recovering Coefficients

Only the two intercept estimates (corresponding to reference level <12 of School) match the numbers in the book. To recover the original estimates in F&T, we need to add the intercepts to each pair of coefficients for each other Schooling levels:

Code 
happy_logit2_vglm_coeff <- attributes(happy_logit2_vglm)$coeff
happy_logit2_vglm_coeff

Log-Likelihood and Model Fit

Add the log likelihood resulting from the first and second parts of (3.5.3), to give the likelihood of the full GLM model:

Code 
log_lik_glm <- logLik(happy_logit1_vglm) + logLik(happy_logit2_vglm)

Computing Saturated Model Likelihood

Code 
happy_male <- happy[which(happy$Sex == "Males"), ]
happy_female <- happy[which(happy$Sex == "Females"), ]

log_lik_saturated <- dmultinom(
  happy_male$n,
  prob = happy_male$n / sum(happy_male$n),
  log = TRUE
) +
  dmultinom(
    happy_female$n,
    prob = happy_female$n / sum(happy_female$n),
    log = TRUE
  )

log_lik_saturated
[1] -57.99938

Deviance

This is the deviance 13.27, as reported in the book:

Code 
2 * (log_lik_saturated - log_lik_glm)
[1] 13.27614

Package citations and references

Code 
packages <- c(
  "conflicted", "dplyr", "Fahrmeir", "tidyr", "VGAM"
)
for (pkg in packages) {
  cit <- citation(pkg)

  # Check if multiple citations
  if (length(cit) > 1) {
    cat("- **", pkg, "**:\n", sep = "")
    for (i in seq_along(cit)) {
      cit_text <- format(cit[i], style = "text")
      cat("    - ", cit_text, "\n\n", sep = "")
    }
  } else {
    cit_text <- format(cit, style = "text")
    cat("- **", pkg, "**: ", cit_text, "\n\n", sep = "")
  }
}
